# SEO-план HouseKit Pro by HotWell.kz

Документ описывает целевой набор SEO-страниц лендинга `housekit.pro`, ключевые
запросы под Google и Яндекс, рекомендуемые `title`/`description`,
структуру H1/H2 и приоритеты выпуска.

> Бренд: **HouseKit Pro** by **HotWell.kz**.
> Продукт: онлайн-программа для проектирования СИП-домов, расчёта СИП-панелей,
> чертежей, PDF-отчётов и спецификаций материалов.
> Локаль: ru-RU. Регион: Казахстан + СНГ.

---

## 1. Принципы

- **Индексируем** только публичные маркетинговые страницы.
- **Не индексируем** `/app`, `/app/*`, `/login`, `/register`, `/invite/*`, `/demo`
  (демо-режим работает как редирект в редактор и не должен попадать в индекс).
- На каждой публичной странице:
  - уникальные `title` (≤ 60 символов рекомендуемо) и `description` (≤ 160);
  - `link rel="canonical"` на самого себя;
  - Open Graph и Twitter Card с превью 1200×630;
  - один H1, далее иерархия H2/H3;
  - JSON-LD: `SoftwareApplication` + `BreadcrumbList`, на FAQ-страницах — `FAQPage`,
    на материалах — `Article`.
- Все маркетинговые страницы должны быть **в `sitemap.xml`** и доступны линками
  из шапки/подвала или из главной.
- В тексте — **естественное** вхождение ключей; H1 содержит главный ключ.

## 5. Технические требования

- **Canonical** — на каждой странице. На главной — `https://housekit.pro/`.
- **`hreflang`** — пока только `ru`. Если появится английская версия — добавить.
- **`robots.txt`** — закрывает `/app`, `/login`, `/register`, `/invite/*`, `/demo`.
- **`sitemap.xml`** — обновлять при выходе каждой новой страницы; не вносить
  приватные маршруты.
- **OG-image** — 1200×630, в фирменном тёмном стиле; для каждой страницы
  допустим уникальный (на старте — общий `/og-image.svg`).
- **Performance бюджет**: LCP ≤ 2.5 c, CLS ≤ 0.1, INP ≤ 200 мс.
- **Mobile-first**: все шрифты ≥ 16 px, тач-таргеты ≥ 44×44.
- **Доступность**: по одному H1, контраст ≥ 4.5:1, явные `:focus-visible`.

### Критерии выпуска (для каждой страницы)

- [ ] Один `<h1>`, ≥ 600 слов уникального контента.
- [ ] Главный ключ вхождение в H1, первые 100 слов, заголовок раздела, URL-slug.
- [ ] Внутренние ссылки: хотя бы по 2 исходящие на смежные SEO-страницы и 1 на главную.
- [ ] Canonical на саму себя.
- [ ] `useDocumentSeo({ robots: "index" })`.
- [ ] OG-тайтл/описание/картинка (на старте — общая `/og-image.png`, позже — уникальная).
- [ ] JSON-LD `SoftwareApplication` + `BreadcrumbList` (+ `FAQPage` где применимо).
- [ ] Страница добавлена в `public/sitemap.xml`.
- [ ] GSC → Inspect URL → Request indexing после деплоя.
- [ ] Lighthouse SEO = 100 на этой странице.

### Порядок выпуска

1. Дизайн-макет общего шаблона SEO-страницы (Hero + секции) — 1 спринт.

- Не добавлять страницы в sitemap до их реального публичного релиза.
